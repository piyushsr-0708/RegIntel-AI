AUTHENTICATION FLOW (VERIFIED WORKING)
================================================================================

1. User opens http://localhost:5173
   → AuthContext checks localStorage for token/user
   → If not found: Show login page
   → If found: Restore session, show dashboard

2. User enters credentials (admin/admin123)
   → POST to /api/auth/login
   → Receive JWT token
   → Store token in localStorage
   → GET to /api/auth/me with token
   → Receive user data
   → Store user data in localStorage
   → Update AuthContext state (setUser)
   → Redirect based on role:
      - head_office → /
      - department → /departments

3. ProtectedRoute checks isAuthenticated
   → If true: Allow access
   → If false: Redirect to /login

4. User refreshes page
   → AuthContext restores session from localStorage
   → User remains logged in
   → Dashboard remains visible

5. User clicks logout
   → Remove token from localStorage
   → Remove user from localStorage
   → Reset AuthContext state (setUser(null))
   → Navigate to /login
