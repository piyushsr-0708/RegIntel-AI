AUTHENTICATION FLOW
================================================================================

User Opens App
    ↓
Check localStorage for token
    ├─ Token exists? → Dashboard
    └─ No token? → Login page
    ↓
User enters credentials
    ↓
POST /api/auth/login
    ├─ Success → Store token
    └─ Failure → Show error
    ↓
GET /api/auth/me
    ↓
Store user info
    ↓
Redirect based on role
    ├─ head_office → /
    └─ department → /departments
    ↓
Dashboard loads
    ↓
All requests include: Authorization: Bearer <token>

================================================================================
API ENDPOINTS USED
================================================================================

POST /api/auth/login
--------------------
Purpose: Authenticate user
Request: FormData (username, password)
Response: { access_token, token_type }
Status: ✅ Operational

GET /api/auth/me
----------------
Purpose: Get user info
Headers: Authorization: Bearer <token>
Response: { id, username, full_name, role, department_id, department_name }
Status: ✅ Operational

All Protected Endpoints
-----------------------
Headers: Authorization: Bearer <token>
Backend validates JWT on every request
Status: ✅ Operational

KNOWN LIMITATIONS
================================================================================

1. No Auto-Logout on Token Expiry
   → Token expires after 8 hours
   → User must manually refresh
   → Future: Add 401 interceptor

2. localStorage XSS Vulnerability
   → Tokens in localStorage
   → Vulnerable to XSS attacks
   → Production: Use httpOnly cookies

3. No Token Refresh
   → Must re-login after 8 hours
   → Future: Implement refresh tokens

4. No Offline Support
   → Requires backend running
   → Future: Add offline mode
